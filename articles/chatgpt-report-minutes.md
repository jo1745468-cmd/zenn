## 「また報告書か」という気持ち、わかります

毎週金曜の夕方、週次報告書を書く30分。
会議が終わるたびに議事録をまとめる時間。
日報を書く5分が、なぜかひどく長く感じる。

内容は頭の中にある。ただ「書く」という作業だけが重い。

ChatGPTを使えば、この「書く」作業を3分に短縮できる。

---

## 議事録が嫌われる本当の理由

議事録が面倒なのは、「記録する」作業だからだ。

会議中にメモを取る。会議が終わったあとに整理する。整理したものを共有する。この3ステップ全部が「誰かのための作業」で、自分には何も残らない感覚がある。

だから後回しになる。後回しにするほど記憶が薄れる。薄れた記憶で書いた議事録は内容が薄い。薄い議事録は「また書いた意味なかった」になる。

解決策は「書く時間を短くする」ことだ。3分で終わるなら、後回しにする理由がなくなる。

## プロンプト1｜メモから議事録を自動生成する

```
以下の会議メモをもとに、議事録を作成してください。

■会議名：【例：週次営業MTG】
■日時：【例：2026年7月23日 10:00〜11:00】
■参加者：【名前または役職】
■メモ：
【ここに走り書きのメモをそのまま貼る】

出力形式：
・決定事項（箇条書き）
・議論になった点（論点と結論）
・次回までのアクション（担当者・期限付き）
・次回議題（あれば）
```

**ポイント：メモは汚くていい**

「山田さん→来週までに見積もり、田中はお客さんに連絡、予算300万、次回8/5」

このレベルのメモでも、議事録の形式に整えてくれる。走り書きをそのまま貼ればいい。

**出力例**

```
■ 決定事項
・来月のキャンペーン予算を300万円に設定する
・新規顧客向けプランのリリース日を8月15日に確定

■ 議論になった点
・予算配分の優先順位（デジタル広告 vs. 展示会）
　→ 今期はデジタル広告に70%を配分することで合意

■ アクションアイテム
・山田：8/1までに見積書を作成・共有
・田中：7/25までに既存顧客Aへ状況確認の連絡
・鈴木：次回MTG（8/5）のアジェンダを7/31までに共有

■ 次回議題候補
・キャンペーン進捗確認
・新規プランのFAQ整備
```

## プロンプト2｜週次報告書を箇条書きから作る

```
以下の今週の活動メモをもとに、週次報告書を作成してください。

■報告対象：【例：営業部長への週次報告】
■今週の活動メモ：
【やったことを箇条書きで並べる】

出力形式：
・今週の成果（数字があれば含める）
・課題と対応策
・来週の予定
・所感（1〜2文）
```

**なぜ「所感」を入れるのか**

報告書を読む上司が一番知りたいのは「現場の肌感」だ。数字は資料を見ればわかる。「今週は競合の動きが気になった」「Aさんの反応が今まで一番良かった」という感覚は、報告書にしか出てこない。

所感を1〜2文入れるだけで、報告書の質が変わる。

**出力例**

```
■ 今週の成果
・新規商談：5件（目標3件、達成率167%）
・提案書送付：3件
・受注：1件（A社、月額15万円）

■ 課題と対応策
【課題】商談後のフォローに時間がかかっている
【対応】フォローメールのテンプレートを今週中に整備する

■ 来週の予定
・B社との2回目商談（火曜午後）
・月次目標の中間確認MTG（木曜）
・新テンプレートの運用開始

■ 所感
A社の受注は競合との比較検討が長引いたが、価格よりサポート体制を評価いただいた。
同じ訴求をC社でも使えそう。
```

## プロンプト3｜日報を1分で終わらせる

```
以下の今日の作業ログから日報を作成してください。

■作業ログ（箇条書きでOK）：
【やったこと・連絡・気づいたことをそのまま並べる】

出力形式：
・実施事項（簡潔に）
・明日の予定
・一言コメント
```

**日報は短くていい**

日報を長く書いても、読む人は3秒しか見ない。

「今日何をやって、明日何をやるか」が30秒で読み取れる日報が最も良い。ChatGPTに「簡潔に」と指定するだけで、それに近づく。

## プロンプト4｜月次レポートの構成を自動で作る

月次レポートは「量」が多いので、構成から考えるだけで30分消える。

```
以下の情報をもとに、月次営業レポートの構成と各項目の内容を作成してください。

■今月の実績（数字）：
【例：新規商談20件、提案書送付12件、受注5件、売上250万円】

■先月比の変化：
【例：受注件数は先月より2件増、売上は30万円減】

■今月の特記事項：
【例：大型案件1件を失注、新商材の提案を開始】

出力：月次レポート本文（見出し・段落構成付き）
```

月次レポートの鬼門は「減った数字」の説明だ。ただ「減りました」と書くだけでは読む側の信頼が下がる。ChatGPTに「要因と対策も含めて」と指定すると、減少を文脈込みで説明してくれる。

## プロンプト5｜会議の事前アジェンダを作る（議事録の前段階）

良い議事録は、良いアジェンダから生まれる。

```
以下の目的で会議のアジェンダを作成してください。

■会議の目的：【例：Q3の営業方針を決める】
■参加者：【役職・人数】
■時間：【例：60分】
■決めたいこと：【箇条書きで】

出力：時間配分付きのアジェンダ（各項目の目的・進行担当含む）
```

アジェンダがあると会議が30%短くなる、という実感がある。議題が明確なので「それ今日決める必要ある？」が事前にわかるからだ。

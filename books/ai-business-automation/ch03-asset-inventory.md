## AIが最初にやること

停止条件を渡してビジネス作成を依頼すると、AIが最初にやることは「棚卸し」だ。

何もない状態から作るより、すでにあるものを使う方が速い。AIはこれを理解しているので、まず「使えるものが何かあるか」を確認しようとする。

実際の実験では、AIは以下を自律的に確認した。

```
確認したもの（実際のログ）：
- ローカルのファイル構成（何のファイルが何本あるか）
- noteへのログイン状態（→ Chrome拡張未接続のため私に確認）
- GitHubへのログイン状態（→ 同上）
- 既存コンテンツの品質・使用可否
```

## 棚卸しの4カテゴリ

AIが評価する資産は大きく4種類に分かれる。

### 1. コンテンツ資産

過去に作ったブログ記事、メモ、資料、スライド、プロンプト集など。

AIがチェックするポイント：
- 完成度（そのまま使えるか、手直しが必要か）
- 独自性（他で手に入らない情報か）
- 需要（誰かが欲しがるか）

### 2. アカウント資産

note、Zenn、Twitter/X、YouTubeなどのプラットフォームアカウント。

AIがチェックするポイント：
- ログイン済みか
- フォロワー数・過去の実績
- そのプラットフォームで何ができるか

### 3. ツール資産

Claude Code、ChatGPT、Canvaなどの契約済みサービス。

AIがチェックするポイント：
- 何が使えるか（無料・有料・API接続）
- 自動化できる工程はどこか

### 4. 知識・経験資産

過去の仕事、専門知識、失敗経験など。これはAIが直接確認できないので、人間が教える必要がある。

## 棚卸しを効率化するプロンプト

依頼の最初にこれを添えると、AIが的確に棚卸しを進める。

```
ビジネスを構築する前に、まず使える資産を確認してください。

私が持っているもの：
- [ローカルファイル：○○フォルダに記事草稿が○本]
- [アカウント：note（ログイン済み）、Zenn（未登録）]
- [ツール：Claude Code（契約中）、ChatGPT Plus（契約中）]
- [経験：○○業界での○年の実務経験]

これらを踏まえて、最も早く利益が出る事業案を提案してください。
```

## 棚卸し後にAIが判断すること

資産の確認が終わると、AIは次の判断をする。

- 「完成に近いものがある → そのまま販売できるか評価する」
- 「素材はあるが未完成 → 制作工数を見積もる」
- 「何もない → ゼロから作る場合と既存購入の比較をする」

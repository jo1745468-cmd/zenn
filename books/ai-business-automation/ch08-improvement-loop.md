## 一度作って終わりにしない

コンテンツを公開した後が本番だ。

「作って終わり」では収益は上がり続けない。PVが集まらなければ改善が必要だし、PVはあっても購入につながらなければ別の問題がある。

これも自動化できる。

## 状態ベースの改善ルール

週次データを見て「今どの状態か」を判断し、状態ごとに決まったアクションを取る。これをAIが自動診断できるように設計する。

| 状態 | 条件 | アクション |
|---|---|---|
| A（流入不足） | PV < 50/週 | タイトル改善・無料記事を追加 |
| B（離脱問題） | PV ≥ 50、クリック率 < 5% | CTA文言・配置を変更 |
| C（購入不足） | クリック率 ≥ 5%、転換率 < 2% | 販売ページの説明文・価格を見直す |
| D（順調） | 購入率 ≥ 2% | 新コンテンツを追加して単価UP |
| E（停滞） | 2週間変化なし | プラットフォーム・ターゲットを見直す |

## 週次診断プロンプト

毎週数字を貼り付けるだけで診断が出るようにしておく。

```
今週の数字を報告します。状態を診断して、今週のアクションを1つ決めてください。

【今週の数字】
- 無料記事PV：[数字]
- 有料ページクリック数：[数字]
- 購入数：[数字]
- 先週からの変化：[増減]

【判断基準】
- 状態A（PV<50）→ タイトル改善・無料記事追加
- 状態B（PV≥50、クリック率<5%）→ CTA改善
- 状態C（クリック率≥5%、転換率<2%）→ 販売ページ改善
- 状態D（転換率≥2%）→ コンテンツ追加・値上げ検討
- 状態E（2週間変化なし）→ 抜本的見直し

診断結果と、今週実行する具体的なアクションを1つ出してください。
```

## 人間の作業は週5分

このループでの人間の作業：

1. 週1回、PV・クリック数・購入数をコピーする（2分）
2. 上記プロンプトに貼り付けてAIに送る（1分）
3. AIの診断結果を確認し、実行を承認する（2分）

合計：週5分。あとはAIが改善案を作成し、変更をpushする。

## 改善の優先順位

どの指標を最初に改善すべきか迷ったときの原則：

```
PVが少ない → 入口を増やす（記事追加・タイトル改善）
PVはあるがクリックされない → 出口を直す（CTA改善）
クリックされるが買われない → 購入ページを直す（説明・価格）
```

上から順番に直す。購入ページを直す前に流入を増やしても効果が出ない。

## 長期の資産化

3ヶ月以上継続すると、検索流入が安定してくる。その段階で：

- 無料記事を5〜10本に増やす
- コンテンツを追加して有料本の単価を上げる
- 別テーマの無料記事→有料本のファネルを追加する

最初の1本を作る工程が自動化できれば、2本目以降は同じフローで量産できる。

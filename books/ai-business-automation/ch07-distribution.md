## プラットフォームの選び方

コンテンツを販売するプラットフォームの選択は、「自動化できるか」で決める。

| プラットフォーム | 自動化レベル | 人間の作業 |
|---|---|---|
| Zenn（GitHub連携） | ★★★★★ | pushの承認のみ |
| note | ★★☆☆☆ | コピペ＋ヘッダー画像＋公開ボタン |
| WordPress | ★★★☆☆ | 初期設定後はAPI連携可 |
| Amazon KDP | ★★☆☆☆ | PDF入稿＋フォーム操作 |

この実験ではnoteで始め、後にZennへ移行した。

**noteを選んだ理由**：
- ログイン済みですぐ始められた
- 500円の有料記事が簡単に作れる
- 日本語ユーザーが多い

**Zennに移行した理由**：
- GitHubにpushするだけで自動公開される
- 人間の作業が「pushの承認1クリック」だけになる
- Claude CodeがMarkdownファイルを直接操作できる

## Zenn×GitHub連携の設定手順

1. GitHubでリポジトリを作成（例：`zenn`）
2. ZennとGitHubアカウントを連携
3. `articles/`フォルダにMarkdownファイルを置く
4. `git push`するとZennに自動反映される

フォルダ構成：

```
zenn/
├── articles/
│   └── 記事スラッグ.md   ← published: true で公開
└── books/
    └── 本スラッグ/
        ├── config.yaml   ← price: 500 で有料化
        └── チャプター.md
```

## 販売設計のポイント

**ファネル（導線）の設計**

```
無料記事（検索流入）
    ↓ CTA（記事末尾のリンク）
有料コンテンツ（500円）
```

無料記事の末尾に必ず有料コンテンツへのリンクを入れる。これがないと、読者は満足して離脱するだけで収益につながらない。

**価格設定**

初回は500円が最適。

理由：
- 読者が「まあいいか」で買える金額
- 内容への不満があっても返金要求されにくい
- 実績（購入者数）が積み上がりやすい

実績が10件以上になったら1,000円への値上げを検討する。

## 公開チェックリスト

AIに以下を確認させてからpushする。

```
公開前に以下を確認してください：

□ プレースホルダーのURL・テキストが残っていないか
□ 有料部分のfree: falseが設定されているか
□ タイトル・サマリーが魅力的か
□ CTAリンクが正しいURLを指しているか
□ 誤字脱字がないか

全項目クリアなら「公開可」と報告してください。
```

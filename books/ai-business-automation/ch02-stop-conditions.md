## 停止条件とは何か

停止条件とは「AIが自動実行を止めて、人間に確認を求めるトリガー」のことだ。

これを定義しておかないと、AIは次の2つの行動パターンに陥る。

**パターンA：何でも確認してくる**
安全側に倒しすぎて、些細なことでも止まる。「このファイル名でいいですか？」「この表現はどうですか？」と毎回聞いてくる。人間の手間が減らない。

**パターンB：何でも実行してしまう**
指示の解釈が広すぎて、公開・送信・購入まで勝手にやってしまう。取り返しのつかない操作が発生するリスクがある。

停止条件はこの中間を設計するものだ。

## 停止条件の3分類

### 分類1：外部への発信・公開

記事の公開、SNS投稿、メール送信、プル リクエストのマージなど、「外の世界に影響が出る」操作はすべて停止条件にする。

理由：一度公開したものは取り消せない。誤った内容が出ると信頼を失う。

### 分類2：費用・契約の発生

購入、新しい有料プランへの登録、APIの課金、ドメイン取得など「お金が動く」操作はすべて停止条件にする。

理由：意図しない費用が発生すると、後から取り消せないことが多い。

### 分類3：他者への影響

顧客への連絡、クライアントへの報告、SNSでの言及など「他の人が関係する」操作はすべて停止条件にする。

理由：誤った情報や不適切な表現が相手に届くと、関係が壊れる。

## 停止条件の書き方

曖昧な書き方では機能しない。具体的な操作名で書く。

**悪い例（曖昧）**
```
危険なことはしないでください
```

**良い例（具体的）**
```
以下の操作は必ず私に確認してから実行してください：
1. 記事・投稿の公開
2. 購入・契約・支払い・新規課金
3. 顧客・第三者へのメッセージ送信
4. 一括削除・データの永久消去
```

## 停止条件をプロンプトに組み込む場所

セッションの最初のメッセージに入れる。後から追加してもAIは忘れることがあるため、最初に渡すのが確実だ。

```
【重要：以下の操作は実行前に必ず確認を求めてください】
- 公開・投稿・送信
- 購入・契約・支払い
- 第三者への連絡

上記以外は確認なしで実行してください。
```

最後の一文「上記以外は確認なしで実行してください」が重要だ。これがないと、AIは念のため何でも確認してくる。
